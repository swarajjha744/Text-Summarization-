#Description:- The Text Summarization Tool is a Natural Language Processing (NLP) application developed as part of Internship Task 1, designed to solve one of the most pressing challenges in today's information-heavy world — the problem of having far too much text to read and too little time to read it. The tool takes any lengthy article, research paper, news report, or document and automatically produces a shorter, concise version that preserves all the important information, and it does this using two fundamentally different NLP techniques: Extractive Summarization and AI-powered Abstractive Summarization. The project is delivered in two forms — a fully interactive web-based interface built with HTML5, CSS3, and vanilla JavaScript, and a standalone Python script that demonstrates the underlying NLP logic using only Python's built-in standard library modules with a live sample article — making it both a functional tool and a clear academic demonstration of NLP concepts. The Extractive Summarization method is the core NLP engine and is implemented entirely from scratch without any third-party libraries like NLTK or spaCy. It works through a four-step pipeline that begins with text pre-processing, where the raw input is tokenized into individual words, converted to lowercase, stripped of punctuation, and filtered through a custom stopword list — stopwords being common English words like "the", "and", "is", and "with" that appear in virtually every document but carry no meaningful information and would otherwise distort the scoring. After cleaning, the algorithm moves to word importance scoring, which is inspired by the TF-IDF (Term Frequency–Inverse Document Frequency) technique widely used in information retrieval and search engine technology; each remaining word is assigned a score based on how frequently it appears in the document, and these scores are normalized so the most frequently occurring word always receives a perfect score of 1.0 while all other words are proportionally ranked below it. Next, the algorithm performs sentence scoring, where every sentence in the document is evaluated by calculating the average importance score of all its meaningful (non-stopword) words — sentences packed with high-importance words receive high scores, while sentences filled with vague or common language score low. Finally, the algorithm selects the top N highest-scoring sentences, where N is determined by the user's chosen summary style, and critically reorders them back into the sequence in which they originally appeared in the document so that the summary reads naturally and logically rather than as a jumbled collection of isolated sentences. The Abstractive Summarization method works on an entirely different principle — instead of extracting existing sentences, it reads and understands the full meaning of the text and then generates completely brand-new sentences that capture the same ideas, exactly the way a human writer would summarize an article after reading it. This is powered by Claude, a state-of-the-art large language model (LLM) developed by Anthropic, accessed through their official REST API endpoint at api.anthropic.com/v1/messages using a structured fetch() request with the correct content-type and authentication headers. The prompt sent to the model is dynamically customized based on the user's selected summary style, meaning the model receives different instructions depending on whether the user wants a single sentence, a short paragraph, or a bullet list, which ensures the output format is always exactly what the user asked for. A defining feature of this tool is that it does not produce one single fixed-length summary — instead, users can choose from five distinct output styles: One Sentence, which compresses the entire article into a single densely informative line ideal for headlines and quick references; Brief Paragraph, which produces a 3–4 sentence natural-language paragraph that covers the main argument and key supporting points and serves as the default balanced mode; 3 Key Points, which breaks the content into three clearly labelled sections each capturing a major theme, useful for presentations and formal reports; Bullet List, which generates five concise takeaways preferred by readers who scan rather than read linearly; and Plain English, which rewrites the content in simple jargon-free language designed for a general audience who may not be familiar with the topic's technical terminology. The web interface is built entirely with native HTML5, CSS3, and ES6 JavaScript without any frontend frameworks, featuring a textarea for pasting text, five capsule-style selector buttons for the summary style, a dropdown for switching between the two summarization methods, a real-time statistics panel that shows original word count, summary word count, and compression percentage calculated as a reduction ratio, a copy-to-clipboard button for exporting results, a sample article loader for instant demonstration, and graceful error handling that shows clear human-readable messages and recommends switching to Extractive NLP if the AI API is unavailable. The Python script mirrors all of this logic using only four built-in modules — re for regular expressions to handle sentence splitting, string for punctuation stripping, heapq for efficient top-N sentence selection using a heap data structure, and collections.defaultdict for frequency counting — and it runs a complete formatted demo on a built-in sample article that covers the topic of artificial intelligence and prints results directly to the terminal in a clean readable layout. On the Python side, the Abstractive mode is available by installing the anthropic package via pip and setting the ANTHROPIC_API_KEY environment variable, after which it connects to the same Claude model and returns AI-generated summaries in any of the five styles. The distinction between these two approaches — statistical sentence extraction based on word frequency versus generative AI that writes fresh natural language — represents two of the most important paradigms in modern NLP, and implementing both in a single cohesive tool with a clean interface and a well-documented Python script demonstrates a solid practical understanding of how Natural Language Processing is applied to solve real-world problems in text analysis, information retrieval, and automated content generation.
